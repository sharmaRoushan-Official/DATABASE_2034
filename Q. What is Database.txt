Q. What is Database?
-------------------------------------------------------------------------
A Database is a place where we store data in an organize way so that we Can:

* Insert Data
* Update Data
* Search and retrieve data easily
* Delete Data
------------------------------------------------------------------------------------
OR

A database is a collections of data stored in a format that can easily be accessed.
-------------------------

Q. Why Database is Needed?
-------------
Without Database:
* Data is stored in files(Excel/Text).
* Searching becomes slow.
* Data be can be lost easily.
* Duplicate record happen.
------------
Database helps to:

* Store data safely.
* Retrieve data quickly.
* Avoid redundancy(duplicate data not allowed)
* Maintain consistency.
-------------------------------------------------------------------------------------------
✅ Types of Database
--------------------------------------------------------------------
A). Relational Database(RDBMS-Relational Database Management System)
Data is stored in tables(rows and columns)
Example:
* MySQL
* PostgreSQL
* Oracle
* SQL Server
------------
B). Non-Relational Database(NoSQL)
Data is stored in the formats like JSON, documents, key-value paris.
Example:
* MongoDB
* Firebase
* Cassandra
* Redis
---------------------------------------------------------------------------------------
Q. What is DBMS?
-----------------------------------------------
DBMS(Database management System) is a software used to manage Databases.

Example:
* MySQL is DBMS
* Oracle is DBMS

DBMS helps to:
* Store data
* Modify data
* Retrieve data
* Manage user access
-----------------------------------------------------------------------------------------
Q. What is RDBMS?
RDBMS(Relational DBMS) stores data in the form of tables with relationships.

Example:

Student Table
Course Table 

Both connected using common columns like student_id.
-----------------------------------------------------------------------------------------------
✅ Basic Database Terminologies.
----------------------------------------
Term		|	Meaning
------------------------------------------------
Table		| Collections of data(like Excel sheet)
------------------------------------------------
Row/Record	| One complete entry.
--------------------------------------------------------
Column/Field	| One property(Name, Age, Salary)
-----------------------------------------------------------
Primary Key	| Unique identifier for each row.
------------------------------------------------------------
Foreign Key	| Links one table to another.
-------------------------------------------------------------
Schema		| Structure/design of database.
-----------------------------------------------------------
Query		| Command to fetch or modify data
--------------------------------------------------------------------------
✅ Keys in Database

Primary key:
-------
* Unique
* Cannot NULL
Example: student_id
------------------------
Foreign Key
* Used to connect two tables.
Example: student_id in another table.

SQL (Stuructured Query lanaguage)
SQL is used to communicate with database.
SQL is used for:
* Create table
* Insert Data
* Update Data
* Delete Data
* Retrieve Data
---------------------------------------------
✅ Basic SQL Commands
1. DDL (Data Definition Language)
Used to define structure.
* Create
* Alter
* Drop
* Truncate
2). DML(Data Manipulation Language)
* Insert
* Update
* Delete
3). DQL(Data Query Language)
* Select
4). DCL(Data Control Language)
* Grant
* Revoke
5). TCL(Transaction Control Language)
* Commit
* Rollback
* Savepoint

Q. What is Normalization?
Normalization is the process of organizing data to reduce duplication.

Forms:
* 1NF - (First Normal Form)
* 2NF - (Second Normal Form)
* 3NF - (Third Normal Form)
* BCNF - (Boyce Code Normal Form)
* 4NF - (Fourth Normal Form)
* 5NF - (Fifth Normal Form)

MySQL Constraints -> Restrictions
-----------------------------
1). NOT NULL
2). UNIQUE
3). DEFAULT
4). CHECK
5). PRIMARY KEY
6). FOREIGN KEY

----------------------------------------
1). NOT NULL-
	PREVENTS INSERTING NULL VALUES.
2). UNIQUE-
	ENSURES VALUES ARE NOT REPEATED(EXMAPLE EMAIL, PHONE)
3). DEFAULT-
	SETS A DEFAULT VALUE IF USER DOES NOT PROVIDE ONE.
4). CHECK-
	VALIDATES CONDITIONS BEFORE INSERTING
5). PRIMARY KEY-
	USED TO UNIQUE IDENTIFY EACH VALUE
6). FOREIGN KEY-
	LINKS TWO TABLES (EXAMPLE- STUDENT - COURSE)

MySQL Like Operator & WildCards
--------------------------------------------
>> SELECT DATA WITH LIKE OPERATORS
-------------------------------------------

Ex:- WHERE NAME LIKE "S%"
-----------------------------------------------------------------------------------------------
WILD CARD CHARACTERS:
	% -PERCENTAGE SYMBOL: REPRESENT ZERO,ONE, OR MULTIPLE CHARACTERS.
 	_ -UNDERSCORE SYMBOL: REPRESENT A SINGLE CHARACTERS
------------------------------------------------------------------------------------------
Pattern		|	Description
-------------------------------------------------------
Like "a%"	| starts with 'a' character
----------------------------------------------------------
Like "%a"	| End with 'a' character
--------------------------------------------------------
Like "%am%"	| have "am" in any position
---------------------------------------------------------
Like "a%m"	| starts with "a" and ends with "m"
-------------------------------------------------------------
Like "_a%"	| "a" in the second position
-------------------------------------------------------------
Like "__a%"	| "a" in the third position
----------------------------------------------------------------
Like "_OY"	| "O" in the second and "Y" in the third position
----------------------------------------------------------------------------------------
syntax:
------------------
select column1, column2, column.........
where name like "pattern"

RegEx- [Regular Expression]
-------------------------------
>> Select Data with regular Expression
----------------------------------------------------------------------
Note: -
	We Found Multiple Result Using Regular Expressions [RegExp]
--------------------------------------------------------------------------------------------
Example-
	Where Name RegExp "Khan% | Poor"
----------------------------------------------------------------------------------
$ -  Last keyword 
| - Next Keyword
--------------------------------------------------------------------------------
Regular Expresison pattern with Descriptions
--------------------------------------------------
Sign	| 	Pattern	| Descriptions
--------------------------------------------------------
^(carot)| "^ra"		| Beginning of the string
------------------------------------------------------------
$ [dollar]| "an$"	| End of string
---------------------------------------------------------------
[....]	| "[rms]"	| Any characters listed between the square bracket.
----------------------------------------------------------------------------
^[....]	| "^[a-z]e"	| Match with in the range and the second position is e.
----------------------------------------------------------------------------------
p1|p2|p3| "tom|john|harry"| Matches any of the patterns p1,p2,p3
---------------------------------------------------------------------------

Syntax:
Select with Regular Expression
-------------------------------------------------------
SELECT COLUMN1,COLUMN2......
FROM TABLE_NAME
WHERE COLUMN_NAME RegExp PATTRN

* Primary key always has unique data.

select Column_list

	from table2

>> UNION and UNIONALL
-----------------------------------------------------------------------------------------------------

✅ What is UNION in MySQL?
Union is used to combine the result of two or more SELECT queries into a single result set.

📌 It combines row vertically(one query result below another)
-----------------------------------------------------------------------------

Syntax of UNION
---------------
SELECT column_list from table1
UNION
SELECT column_list from table2;
--------------------------------------------------------------------------
✅ Rules of UNION

✅ Rule 1:
Each SELECT must have the same number of columns

✅ Rule 2:
The column data type should be compactable

✅ Rule 3;
Columns name in output come from the Select query

✅ Rule 4:
UNION removes duplicate rows automatically.
----------------------------------------------------------------------------------


✅ What is UNION ALL?
>> UNION ALL is same as UNION but:

 ✅It keeps duplicate rows
 ✅ Faster than UNION(because no duplicate check)
---------------------------------------------------------------------------------------------------

✅ UNION vs UNION ALL
---------------------------

Feature			|UNION	|	UNION ALL	|
----------------------------------------------------------------
Removes duplicates	|✅YES	|❌ NO			|
------------------------------------------------------------
Keeps duplicates	|❌ NO	|✅ YES			|
-----------------------------------------------------------------
Performance		|Slower	|Faster			|
-----------------------------------------------------------------------
Uses Sorting		| Yes	| NO			|
------------------------------------------------------------------------------------

------------------------
The `PERIOD_ADD(P, N)` function in MySQL adds **N months** to period **P**, where P is in the format **YYMM** or **YYYYMM**.**Result: `16321711`**

Here's how MySQL processes it:

| Step | Detail |
|---|---|
| Input Period | `16321618` |
| Extract Month | `16321618 % 100` = **18** |
| Extract Year | `16321618 // 100` = **163216** |
| Add 5 months | `18 + 5` = **23 months** = 1 year + 11 months |
| New Year | `163216 + 1` = **163217** |
| New Month | **11** (November) |
| **Final Result** | **`16321711`** (YYYYMM format) |

--------------------------------------------------------------

> **Note:** `PERIOD_ADD` is not date-range validated — it works purely arithmetically on the YYYYMM format. The input month `18` is invalid in real calendar terms, but MySQL still processes it mathematically.
